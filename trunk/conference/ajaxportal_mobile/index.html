Описание приложения Social Network
Приложение Social Network состоит из трех страниц: главной (синяя), фотографии (зеленая) и сообщения (оранжевая). Переходы между страницами осуществляются при помощи кнопок меню. Это статическая демоверсия (правильнее было бы сказать - HTML-прототип мобильного приложения некой социальной сети). Интерфейс сделан с расчетом отображения приложения на относительно маленьком экране с небольшим разрешением. В качестве аналога можно рассматривать приложение http://m.odnoklassniki.ru.

P.S. Вы можете загрузить Social Network приложение в отдельном окне http://sokolovbook.narod.ru/en/socnet/index.html

Зачем эта демоверсия?
Это демоверсия Ajax Portal показывает два способа превращения приложения Social Network в портлет:
1) В левой колонке помимо этого описания размещено приложение Social Network, которое загружено в IFrame (является абсолютно независимым субокном браузера в главной Интернет странице). В использовании IFrame нет ничего нового и это стандартный прием.
2) В правой колонке загружено это же приложение в режиме использования портала, т.е. в главной странице (синей) существует код проверяющий доступно ли API портала. Оно сможет быть доступно, только если загруженное приложение станет частью портальной страницы. Это осуществляется использованием библиотеки ajax4all. Если API портала найдено и загружается главная (синяя) страница приложения, то происходит загрузка двух других страниц (зеленой и оранжевой) в отдельных портлетах. Навигационные кнопки при этом скрываются. Далее будет доступна персонализация приложения Social Network средствами портала (позиция портлета, заголовок портлета, состояние скрыт/свернут/нормальное). Это концептуальное демо и в нем персонализация еще не реализована. Это будет сделано позже.

Мобильное приложение Social Network может само определять наличие Ajax Portal и разворачивать свои составные части для максимального использования поверхности окна браузера, т.е. разработчики имеют возможность разрабатывать одно приложение пригодное как для обычных компьютеров так и для мобильных устройств без серьезных изменений. Есть еще одна возможность - саморазворачивающиеся порталы, но это совсем другая история.
